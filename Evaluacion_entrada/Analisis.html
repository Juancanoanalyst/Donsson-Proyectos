1.Identificacin de variables relevantes para estimar el inventario ideal

Para construir un modelo de inventario ideal, se consideran variables cuantitativas y cualitativas que reflejan el comportamiento de la demanda, la criticidad del producto y la confiabilidad de los datos. A continuacin, se listan y justifican:

Variables cuantitativas

  • Demanda promedio mensual Refleja la cantidad promedio vendida por mes. Es la base para estimar el consumo esperado y calcular el stock necesario.

  • Desviacin estndar de la demanda Mide la variabilidad en la demanda. Es esencial para calcular el inventario de seguridad que cubre incertidumbre.

  • Costo unitario Permite priorizar productos en funcin de su impacto financiero. Ayuda a aplicar polticas diferenciadas de inventario (por ejemplo, stock mnimo ms bajo para productos costosos).

  • Ventas totales Ayuda a identificar productos de alto impacto econmico, tiles para anlisis tipo ABC o decisiones estratgicas de stock.

Variables cualitativas

  • Validez del producto (SKU vlido) Se valid previamente la estructura del cdigo SKU para asegurar que los productos considerados sean reales y operativos.

  • Estado del costo (costo vlido/sospechoso) Se filtraron los productos con valores de costo irrealmente bajos, los cuales podran distorsionar el modelo.

  • Rotacin (historial de ventas) Se excluyeron productos sin ventas en el perodo analizado, ya que no permiten estimar demanda ni justificar stock.

2. Propuesta de modelo de inventario ideal y justificacin

Para estimar el inventario ideal por SKU, se utiliz un modelo clsico basado en la frmula:

Inventario ideal = Demanda promedio mensual + Inventario de seguridad

El inventario de seguridad se calcula a partir de la desviacin estndar de la demanda, considerando un nivel de servicio deseado del 95%, lo cual corresponde a un valor Z = 1.65 en una distribucin normal estndar. Dado que el lead time (tiempo de reabastecimiento) no fue proporcionado, se asumi un lead time de 1 mes.

La frmula completa aplicada fue:

  • Demanda distribuida normalmente, segn el histrico mensual.
  • Lead Time = 1 mes, asumido por simplicidad.
  • Nivel de servicio objetivo = 95%, aplicando un Z = 1.65.
  • Los productos sin ventas fueron excluidos previamente del anlisis, dado que no se puede estimar una demanda futura ni aplicar inventario objetivo.
  • Se utilizan columnas generadas en el anlisis exploratorio: demanda_promedio y desviacion_demanda.

3. Estimacin del nivel de servicio alcanzado

El nivel de servicio representa la probabilidad de tener disponible un producto cuando el cliente lo solicita. Para calcularlo, se utiliz el modelo de inventario ideal propuesto en el punto anterior como referencia de stock.

Dado que no se cuenta con datos reales de inventario disponible, se asumi que el inventario actual es igual al inventario ideal, lo cual corresponde a un escenario donde la empresa ha logrado implementar las polticas de stock propuestas.

Para cada producto, se aplic la frmula:

Supuestos clave:

  • El inventario actual = inventario ideal Dado que no tenemos inventario incial
  • La demanda sigue una distribucin normal
  • Se prioriza una cobertura del 95% de las solicitudes

Conclusiones y recomendaciones generales

Durante el anlisis del inventario histrico de filtros se realizaron procesos de limpieza, transformacin y evaluacin analtica orientados a proponer un modelo de inventario ideal y estimar el nivel de servicio. A continuacin se resumen los principales hallazgos y recomendaciones:

Exploracin y limpieza de datos (EDA)

  • Errores en unidades vendidas: Se identificaron valores negativos en las ventas mensuales, los cuales fueron corregidos por ser inconsistentes con la lgica de ventas reales.

  • Anomalas en cdigos SKU: Se detect una mezcla de formatos en la columna producto, incluyendo errores de digitacin y registros no vlidos (e.g., "FILTROS PARA MUESTRA COMERCIAL", "SERV11", "ENERGA ELCTRICA"). Estos fueron eliminados para garantizar la consistencia del anlisis.

  • Costos unitarios sospechosos: Algunos productos presentaban un costo_unitario extremadamente bajo (incluso igual a $1), inconsistente con los precios del mercado. Para estos casos, se construy un dataframe de productos sospechosos, que se recomienda enviar a los equipos de contabilidad o ventas para su revisin y validacin.

  • Productos sin rotacin: Se identificaron ms de 2.200 productos sin ventas durante el ltimo ao. Se recomienda analizar si estos productos deben ser descontinuados, promocionados o ajustados en inventario para evitar costos de almacenamiento innecesarios.

Modelado de inventario y anlisis estratgico

  • Modelo de inventario ideal: Se utiliz una frmula basada en la distribucin normal para calcular el inventario ptimo por SKU. Esto permite cubrir la demanda mensual esperada con un nivel de servicio del 95%, considerando la variabilidad en las ventas.

  • Nivel de servicio estimado: Bajo el modelo propuesto, todos los productos alcanzaran un nivel de servicio terico de ~95%, ideal para operaciones que requieren alta disponibilidad. Se sugiere usar esta mtrica como base para monitoreo continuo.

  • Clasificacin ABC (Pareto): Se construy una curva de Pareto que revel que el 16% de los productos representan el 80% de las ventas. Esta informacin permite implementar una poltica de inventario tipo ABC, donde los productos A tengan mayor control y prioridad de stock.

Recomendaciones generales

  1. Auditar y depurar SKUs no vlidos: Mantener una estructura estandarizada evita errores en anlisis y en operaciones logsticas.
  2. Crear alertas automticas de costos sospechosos, con base en valores de mercado y mrgenes.
  3. Implementar inventarios diferenciados por criticidad del producto (modelo ABC).
  4. Establecer un proceso de revisin peridica para productos sin rotacin.
  5. Automatizar el clculo de inventario ideal y nivel de servicio, integrndolo a un dashboard gerencial o sistema ERP.

Este anlisis proporciona un marco slido para tomar decisiones informadas que mejoren la eficiencia del inventario, reduzcan quiebres de stock y optimicen los costos de operacin.
